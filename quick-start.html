Getting Started with the Camcloud REST API

This short guide walks you through generating an API key, authenticating in Swagger, discovering your account and camera IDs, and pulling your first data. It’s a rapid on-ramp—within 5 minutes you should be programmatically streaming, recording and managing cameras.

  1. Create an API Key → In the Camcloud portal open Settings › API Keys.
  2. Authenticate in Swagger → Navigate to https://api.hostedcloudvideo.com, click Authorize and paste your key.
  3. Grab your accountID by executing AuthorizeV2.
  4. List cameras with ListCamerasV1.
  5. Fetch a snapshot via getSnapshotForCameraV1 supplying a cameraID.

Authentication & Authorisation

The API is secured with a JSON Web Token (JWT) that you obtain via the AuthorizeV2 endpoint. The token is valid for 24 hours by default; simply refresh the token before expiry to avoid service disruption.

# Request
POST /authorizeV2
Content-Type: application/json
{
  "apiKey": "YOUR-API-KEY"
}

# Response
{
  "token": "eyJhbGciOi...",
  "expiresIn": 86400
}

Pass the token in the Authorization: Bearer <token> header on subsequent requests.

Core Camera Operations

The table below lists the most frequently used camera endpoints.

EndpointMethodDescription
ListCamerasV1GETReturns every camera in the given account
AddCameraV2POSTProvision a new camera (ONVIF or custom RTSP)
UpdateCameraV2PATCHUpdate a camera’s name, timezone, recording mode, etc.
DeleteCameraV1DELETEDe-provision (remove) a camera
getSnapshotForCameraV1GETGrab a JPEG snapshot at the current moment

Live Streaming & Playback

For browsers or native apps use the getStreamUrlForCameraV1 endpoint to retrieve an HLS or RTMP URL. A signed URL is returned—pass it directly to your player of choice (e.g. Video JS, ExoPlayer, VLC).

Event Webhooks

Receive instant push notifications for motion, tamper, disconnect and cloud recording events. Configure webhooks via the portal or CreateWebhookV1. The payload includes identifiers, timestamps and a signed snapshot URL.
